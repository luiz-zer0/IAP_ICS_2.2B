Common Issues with Manual Moodle Installation

Installing Moodle without XAMPP can be challenging. Below are the most common issues and how to fix them.

Step-by-Step Troubleshooting Guide

1Apache Configuration

Ensure Apache is properly configured to serve Moodle from your htdocs directory:

<VirtualHost *:80> DocumentRoot "C:/Apache24/htdocs" ServerName localhost <Directory "C:/Apache24/htdocs"> Options Indexes FollowSymLinks AllowOverride All Require all granted </Directory> </VirtualHost>
Important: After making changes to Apache configuration, restart the Apache service.

2PHP Configuration

Make sure PHP is properly installed and configured with Apache:

  • Check that PHP is added to your system PATH
  • Verify that the PHP module is loaded in Apache
  • Enable necessary PHP extensions in php.ini: extension=curl extension=fileinfo extension=gd2 extension=mbstring extension=exif extension=xmlrpc extension=soap

3Directory Permissions

Moodle needs write permissions to certain directories:

  • Right-click on your Moodle directory
  • Select Properties > Security
  • Ensure that the user account running Apache has modify permissions
  • Specifically, the moodledata directory needs full write permissions

4Database Configuration

If you're using MySQL or MariaDB without XAMPP:

  1. Ensure the database service is running
  2. Create a database and user for Moodle
  3. Verify that the PHP MySQL extension is enabled
  4. Test the database connection with a simple PHP script

5URL Rewriting

For clean URLs to work, enable mod_rewrite in Apache:

LoadModule rewrite_module modules/mod_rewrite.so

And ensure AllowOverride is set to All in your Apache configuration.

Quick Checklist

  • ✓ Apache service is running
  • ✓ PHP is properly configured with Apache
  • ✓ Required PHP extensions are enabled
  • ✓ Database server is running
  • ✓ Directory permissions are set correctly
  • ✓ mod_rewrite is enabled in Apache
Success Tip: After completing these steps, try accessing Moodle at http://localhost. If you still encounter issues, check the Apache error logs for detailed information.
